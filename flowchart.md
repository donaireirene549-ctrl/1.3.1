## 未解決問題清單

### 🔴 關鍵問題（阻擋流程）

| # | 問題 | 影響範圍 | 說明 |
|---|------|---------|------|
| 1 | **Gemini API Key 無效** | 店號/地址讀取 | 提供的 key `AQ.` 開頭，非 Google AI Studio 標準格式 `AIza`，API 呼叫無回應。多數訂單的店號和地址藏在訊息圖片中，必須有可用的視覺 AI 才能讀取。 |
| 2 | **Google Sheets 寫入失敗** | 資料寫入 | Playwright 開啟的是獨立瀏覽器，未登入 Google 帳號。需要：(a) 手動登入 Google，或 (b) 設定 Google Sheets API Service Account，或 (c) 使用系統瀏覽器 profile。 |
| 3 | **圖片中的店號/地址無法提取** | 7-11/FM/HOME | 訊息中的店號和地址經常以圖片形式存在（店家截圖、門牌照片），無可用的視覺 API 時完全無法處理。本次 20 筆中有 8 筆因此失敗。 |

## 解決方案建議

### 問題1+3：Gemini API → 改用正確的 API Key
```
請提供 Google AI Studio 的 API Key（AIza 開頭）
取得方式：https://aistudio.google.com/apikey
```

### 問題2：Google Sheets 寫入 → 三種方案
```
方案A（最簡單）：手動登入
  → 腳本會暫停 60 秒等待手動登入 Google

方案B（最穩定）：Google Service Account
  → 需到 Google Cloud Console 建立服務帳號
  → 下載 JSON 金鑰檔
  → 將試算表分享給服務帳號 email

方案C（中間路線）：使用系統瀏覽器 profile
  → 用已登入 Google 的 Edge/Chrome profile 開啟
```
